## Hoe is het gegaan?
Het afgelopen semester was intensief, maar vooral ontzettend leerzaam. Ik heb aan veel verschillende opdrachten gewerkt, veel ervaring in code opgedaan en vooral ook plezier gehad! Het hoogtepunt was voor mij de meesterproef, het bouwen van een uitgebreide, complexe en volledig interactieve tool.

## Waar ik trots op ben
Ik ben er enorm trots op dat we met de meesterproef met een groep van vijf zo’n complete applicatie hebben neergezet. Omdat ik aan veel verschillende onderdelen van de animatie tool heb meegewerkt, en daarnaast met alle vakken zoveel verschillende dingen heb gedaan, heb ik echt het gevoel dat ik allround ben gegroeid. Maar natuurlijk was niet alles tijdens deze minor individueel en ik ben trots op de samenwerking tijdens zowel de hackathon, een project dat maar 4 dagen duurde, en van de meesterproef, die wel langer duurde maar ook ontzettend uitgebreid was. 

## Wat was lastig & grootste leermomenten
Tijdens de meesterproef was het in het begin best lastig om overzicht te houden in zo’n groot project met meerdere developers. Een van mijn leermomenten (mede door de kennis van teamgenoot Kerr) was dan ook de ontdekking van het MVC-patroon: het strikt gescheiden houden van data, interface en interactie. Toen ik die structuur eenmaal doorhad, werd de code veel logischer. Daarnaast was het werken met GSAP compleet nieuw voor mij. Het had een leercurve, maar het is gaaf om te zien wat er allemaal mogelijk is met zo'n library.

## Wat viel mee
De samenwerking og taakverdeling vielen me 100% mee. Waar groepswerk soms stroef kan lopen, hadden we nu een hele fijne dynamiek, dit zorgde ervoor dat het hele project een fijne ervaring was met minimale stress.

## Persoonlijke Leerdoelen
01. Componenten & dynamische layouts
Tijdens de Meesterproef ben ik hier al goed mee bezig geweest, omdat een tool-interface een totaal andere denkwijze en layout vereist dan een standaard website. Maar ik heb dit leerdoel vooral toegepast in mijn persoonlijke blog. Door Astro te gebruiken kon ik mijn interface opbouwen uit losse, herbruikbare componenten. Om mijzelf uit te dagen ben ik aan de slag gegaan met CSS Subgrid om een speelse, asymmetrische lay-out te maken met bento tiles. Het resultaat is een dynamische structuur zonder dat de code een chaos werd.

02. Toegankelijkheid by default
Bij de Meesterproef was toegankelijkheid geen harde eis, al hebben we daar wel rekening gehouden met dingen als aria-labels. In mijn blog heb ik dit leerdoel wel uitgebreider aangepakt. Ik heb mezelf aangeleerd om eerst robuuste, native HTML te schrijven voordat ik ook maar naar CSS of JS keek. Tijdens het bouwen heb ik regelmatig mijn muis fysiek weggelegd om te testen of alles via de 'Tab'-toets bereikbaar was. Ook heb ik VoiceOver gebruikt om de site te navigeren; dat gaf me direct de bevestiging dat de onderliggende logica klopt, ook als je het scherm niet kunt zien.

03. Team & leiderschap
Vooraf was mijn doel om te oefenen met delegeren en loslaten. Dit ging eigenlijk verrassend natuurlijk, ook omdat ik in een team zat met gemotiveerde en serieuze mensen. Ze hadden weinig aansporing nodig, waardoor ik niet in de valkuil stapte om de 'kartrekkende leader' te worden. We werkten echt samen als gelijkwaardige teamgenoten. Wat deze samenwerking zo prettig maakte, was onze open communicatie. We spraken uit waar we tegenaan liepen og gaven irritaties tijdig aan. Mijn belangrijkste les hieruit: spreek eventuele verwachtingen en irritaties uit, want als je iets niet aangeeft kan je iemand ook niks kwalijk nemen.

## Highlights Weekly Nerd Sessies
Mijn favoriete Weekly Nerd was Nils Binder. Ik ben sowieso fan van Nils als persoon, hij is op een authentieke manier anders dan de andere docenten en dat vond ik leuk. Daarnaast vond ik het nuttig om van hem te leren over de mogelijkheden met CSS. Dingen als subgrid en view transitions heb ik uit zijn verhaal meegenomen.

## Highlights van alle vakken
Ik vond de verschillende vakken tijdens deze minor allemaal zo verschillend dat ik ze bijna niet met elkaar kan vergelijken. Als ik terugdenk aan de afgelopen 5 maanden weet ik wel dat ik CSS een heel leuk vak vond, lekker priegelen zonder de ingewikkeldheid van javascript. Tegelijkertijd staat de meesterproef ook hoog op mijn lijstje, juist omdat die gebouwd is op zoveel ingewikkelde javascript en dat ook weer heel leerzaam was.

## What’s Next?
Ik ben nog op zoek naar een stage voor volgend jaar, of dat in UX/UI of webdevelopment gaat zijn weet ik nog niet, ik blijf beiden interessant vinden! Ik weet in ieder geval zeker dat als ik een stage in web development ga doen, dat ik veel ga hebben aan de ervaring die ik heb opgedaan tijdens deze minor en alle nieuwe dingen die ik heb geleerd.

Op korte termijn ga ik mijn nieuw verworven skills in ieder geval inzetten om mijn blog website om te bouwen tot een portfolio, en ik ga ervan genieten dat ik daarvoor geen harde deadline op DLO heb! Even rust in de tent ;)
