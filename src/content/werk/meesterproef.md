Tijdens de meesterproef werkten wij in een team van 5 personen aan een applicatie voor een echte opdrachtgever. Onze opdarchtgever, Marjolijn, onderzoekt in haar project (Inter)facing the Hybrid City hoe de hedendaagse stad kan worden verkend en verbeeld als een hybride omgeving waarin fysieke en digitale lagen steeds meer met elkaar verweven raken. Binnen dit onderzoek ontbreekt echter een praktische manier om typografische animaties te maken, te testen en daarmee te experimenteren. Daarom was het onze uitdaging om een animatietool te ontwikkelen die dit mogelijk maakte. De tool moest aansluiten op haar onderzoek en gebruikers in staat stellen om op een toegankelijke en flexibele manier typografische animaties te creëren en te verkennen.

## Mijn inbreng
Ik heb tijdens dit project aan veel verschillende onderdelen van de tool gewerkt. Van het toevoegen van functionaliteiten (zoals rotate, scale X, text casing en SplitText) en onderzoeken hoe de gebruiker zelf tekst kan invoeren en selecteren tot het toevoegen van een reset canvas knop en een knop om een animatie property te verwijderen. Daarnaast heb ik ook bijgedragen aan de styling, en dan vooral het toevoegen van animaties op bijvoorbeeld de reset canvas knop, de delete property knop en de uitklapbare details. Al met al heb ik overal wat gedaan en heb ik daardoor tijdens het hele project veel variërend werk verricht.

## Mijn inzichten
Dat ik overal wat gedaan heb vind ik heel nuttig, hierdoor heb ik van veel onderdelen iets meegekregen en meer ervaring opgedaan zonder dat dit op 1 specifiek gedeelte gefocust was. Die ervaring en de daarbij opgedane kennis alleen al vind ik een mooi inzicht om mee te nemen uit dit project.

Ik heb niet eerder een applicatie gebouwd die zo interactief is en zoveel mogelijkheden heeft, daardoor heb ik tijdens dit project heel veel geleerd. De inbreng van Kerr zijn kennis heeft hier ook zeker aan bijgedragen. Een van de inzichten die ik heel leerzaam vond, is hoe belangrijk het in zo’n groot project is om in de code de data, de interface en de interactie gescheiden te houden (MVC patroon). Door die code goed te scheiden og duidelijk te hebben welke file wat doet, had ik veel meer inzicht en begrip in de werking van de code. 

Naast heb ik voor het eerst met GSAP, en daarmee überhaupt voor het eerst met een library, gewerkt. Ik vond het heel nuttig om daarmee te leren werken en ik vind het gaaf hoeveel met GSAP mogelijk is!

Wat ik ook heel leerzaam vond is om met een groep te werken aan zo’n uitgebreid project. Er is flink wat communicatie en goede samenwerking nodig om dat soepel te laten verlopen, dus ook dat vond ik erg leerzaam.
