Dit was zonder twijfel het meest uitdagende CSS-project dat ik tot nu toe heb gedaan. Ik maakte voor dit vak een interactief control panel, zonder JavaScript, waarmee ik een digitale speelkaart kon manipuleren.

## Nieuwe Inzichten
Omdat ik tijdens dit vak de harde eis kreeg om te werken zónder classes of ID's, maar ik deze wel echt nodig had tijdens het opbouwen van de speelkaart, heb ik in eerste instantie wel gebruik gemaakt van een soort classes. 

## Hoogtepunten

### Code
Het hoogtepunt in mijn code was toen ik met behulp van container style queries de speelkaart tot leven kon laten komen en ik kon wisselen tussen de speelkaart van de koningin en de koning, beiden met eigen animaties.

### Ontwerp
Ik ben heel trots op de easter eggs en de speelse details in mijn ontwerp. Ik heb mijn best gedaan om de speelkaart tot leven te laten komen: van de koplampen van de autootjes die langzaam oplichten als je de 'nacht'-slider omhoog schuift, tot de dwarrelende iconen en het zwaard dat begint te hakken als je de juiste knoppen combineert. Het voelt daardoor als een levendig geheel in plaats van een statische pagina.

### Werkwijze
Het hoogtepunt in mijn werkwijze was de keuze om het project in twee fases op te splitsen. In plaats van direct de "no-class" regel toe te passen en in de knoop te raken, heb ik alles eerst opgebouwd met tijdelijke data-naam attributen. Pas toen de hele lay-out en alle animaties stonden, ben ik stap voor stap de HTML gaan ombouwen naar semantische tags (zoals `<aside>`, `<figure>` en `<footer>`) en ben ik de CSS gaan nesten. Dit zorgde ervoor dat ik tijdens het bouwen overzicht bleef houden.

## Verder ontdekken
Dit project heeft me laten zien dat CSS een complete, logische programmeertaal is waarmee veel meer mogelijk is dan ik dacht. Ik ben heel benieuwd naar andere nieuwe CSS features, zoals Scroll-driven animations, om te kijken of ik de interacties die ik nu met checkboxes heb gemaakt, in een volgend project aan het scrollgedrag van een gebruiker kan koppelen.
